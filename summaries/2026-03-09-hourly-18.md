### 09:17-09:45 - 多代理团队框架深度学习 ✅

**核心概念：**

| 概念 | 说明 |
|------|------|
| CONDUCTOR | 团队领导Agent，负责分解任务、调度角色、仲裁冲突 |
| Roles | 专业子代理，有明确使命和输出 |
| Blackboard | 团队内部共享状态文件 |
| Event Bus | 跨团队事件驱动协作 |
| Tool Bootstrap | 注入给子代理的工具/质量规则 |

**运行模式：**
- **Full Pipeline**: CONDUCTOR → 并行dispatch → 串行阶段 → 最终决策
- **Event-Driven**: EventBus路由 → CONDUCTOR → 定向specialist → 事件写回
- **Reactive**: CONDUCTOR → 并行分析 → 快速决策

**示例团队：**
- ecommerce-team: 市场分析、定价策略
- data-collection-team: 数据采集
- content-team: 内容创作
- intelligence-team: 情报分析

**关键规则：**
1. 并行优先 - 独立子任务必须并行
2. 置信度标注 - 所有结论标 HIGH/MEDIUM/LOW
3. 来源标注 - 无来源数据 = 无效
4. 红队自检 - 输出前自我批判
5. 数据优先 - 冲突时量化数据胜出
