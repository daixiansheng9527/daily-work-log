## 多代理团队建设

### media-creator-team（自媒体内容创作团队）
- **创建时间**：2026-03-09 21:40
- **路径**：`teams/media-creator-team/`
- **团队规模**：9人
- **目标**：专业化的自媒体内容创作，从选题到发布的全流程

#### 团队角色架构

| 角色 | 职责 | 模型 |
|------|------|------|
| **CONDUCTOR（内容总监·调度）** | 接收任务、分配资源、协调流程 | glm-4.7-cc |
| **IDEATOR（创意总监·选题）** | 趋势分析、选题策划、创意发散 | grok-4.1-fast |
| **RESEARCHER（资深编辑·调研）** | 深度调研、资料收集、大纲构建 | glm-4.5-air:free |
| **WRITER（资深文案·撰写）** | 撰写初稿、平台适配、叙事结构 | glm-5-cc |
| **REVIEWER（主编·审核）** | 内容审核、事实核查、质量把关 | gpt-5.4 |
| **DESIGNER（视觉总监·设计）** | 封面设计、配图制作、排版布局 | grok-imagine-1.0 |
| **PUBLISHER（运营总监·发布）** | 标题优化、发布时间、多平台分发 | glm-4.7-cc |
| **ANALYST（数据分析师·复盘）** | 数据跟踪、效果分析、复盘总结 | glm-5-cc |
| **ARCHIVIST（知识官·归档）** | 知识归档、经验沉淀、方法论提炼 | glm-4.5-air:free |

#### 工作流
```
CONDUCTOR → IDEATOR → RESEARCHER → WRITER → REVIEWER (质量门)
                                           ↓ (返工)
                                         DESIGNER → PUBLISHER (质量门) → ANALYST → ARCHIVIST
```

#### 质量门（Quality Gates）
1. **内容审核**：REVIEWER 审核 → 通过→DESIGNER，不通过→WRITER
2. **发布检查**：PUBLISHER 检查 → 通过→ANALYST

#### 设计原则
- **并行优先**：独立任务可并行执行
- **质量门**：两个关键检查点（内容审核、发布检查）
- **降级策略**：单点失败不阻塞整体流程
- **模型优化**：根据任务复杂度选择最优模型
- **成本控制**：简单任务使用免费模型节省成本
